INDRA: Integrated Network and Dynamical Reasoning Assembler

INDRA is an automated model assembly system interfacing with NLP systems and databases to collect knowledge, and through a process of assembly, produce causal graphs and dynamical models.

INDRA utilises multiple natural language processing systems and structured databases

INDRA DB: The INDRA Database

The INDRA Database is a framework for creating, maintaining, and accessing a database of content, readings, and statements. This implementation is currently designed to work primarily with Amazon Web Services RDS running Postgres 9+. Used as a backend to INDRA, the INDRA Database provides a systematic way of scaling the knowledge acquired from other databases, reading, and manual input, and puts that knowledge at your fingertips through a direct Python client and a REST api.

Searching the INDRA Database

You can search the INDRA Database for statements here: https://db.indra.bio/
Using the search interface one can search for statements involving e.g. specific genes or chemicals.
Here is an example of searching for statements where ACE2 is object while returning a maximum of 100 results:
https://db.indra.bio/statements/from_agents?object=ACE2&max_stmts=100&format=html

The INDRA Network Search

The INDRA Network Search service allows you to do pathfinding in a graph representation of a snapshot of the INDRA database.

Search Options

The pathfinding can be done between two nodes or it can be an open ended search downstream or upstream from a node. Several search options are available such as:

  • Signed search: e.g. search for upregulations of a certain gene.
  • Namespace restricted search: e.g. only search for genes and chemical entities.
  • Search only over specific statement types.
